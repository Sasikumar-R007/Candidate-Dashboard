# Environment Variables Checklist for Render Backend

## Required Environment Variables

### ✅ Core Configuration

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `NODE_ENV` | ✅ Yes | Environment mode | `production` |
| `NODE_VERSION` | ✅ Yes | Node.js version | `20.10.0` |
| `DATABASE_URL` | ✅ Yes | PostgreSQL connection string | `postgresql://user:pass@host:port/db` |
| `FRONTEND_URL` | ✅ Yes | Frontend application URL | `https://yourdomain.com` |
| `SESSION_SECRET` | ✅ Yes | Secret for session encryption | Auto-generated by Render |

### ✅ Authentication (Google OAuth)

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `GOOGLE_CLIENT_ID` | ⚠️ If using OAuth | Google OAuth Client ID | `123456789-abc.apps.googleusercontent.com` |
| `GOOGLE_CLIENT_SECRET` | ⚠️ If using OAuth | Google OAuth Client Secret | `GOCSPX-xxxxx` |
| `GOOGLE_CALLBACK_URL` | ⚠️ If using OAuth | OAuth callback URL | `https://backend.onrender.com/api/auth/google/callback` |

### ✅ Email Service (Resend)

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `RESEND_API_KEY` | ⚠️ If using email | Resend API key | `re_xxxxx` |

### ⚠️ Optional Variables

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `PORT` | ❌ No | Server port (auto-set by Render) | `10000` |
| `BACKEND_URL` | ❌ No | Backend URL (if needed) | `https://backend.onrender.com` |

## How to Check Environment Variables in Render

### Step 1: Access Environment Tab
1. Go to https://dashboard.render.com
2. Click on your backend service (`staffos-backend`)
3. Click on **"Environment"** tab

### Step 2: Verify Each Variable

Check each variable from the list above:
- ✅ Present and has a value
- ❌ Missing - needs to be added
- ⚠️ Empty - needs to be set

### Step 3: Add Missing Variables

1. Click **"Add Environment Variable"**
2. Enter **Key** and **Value**
3. Click **"Save Changes"**
4. Backend will automatically redeploy

## Environment Variable Format Guidelines

### DATABASE_URL
```
✅ Correct: postgresql://username:password@hostname:port/database
❌ Wrong: postgres://... (missing 'ql')
❌ Wrong: postgresql://...?sslmode=require (Render handles SSL automatically)
```

### FRONTEND_URL
```
✅ Correct: https://yourdomain.com
✅ Correct: https://www.yourdomain.com
❌ Wrong: https://yourdomain.com/ (trailing slash)
❌ Wrong: http://yourdomain.com (use https)
```

### GOOGLE_CALLBACK_URL
```
✅ Correct: https://your-backend.onrender.com/api/auth/google/callback
❌ Wrong: http://... (use https)
❌ Wrong: Missing /api/auth/google/callback
```

## Quick Verification Script

You can add this to your backend to verify env vars (for debugging):

```typescript
// server/env-check.ts (temporary file for verification)
const requiredVars = [
  'NODE_ENV',
  'DATABASE_URL',
  'FRONTEND_URL',
  'SESSION_SECRET'
];

const optionalVars = [
  'GOOGLE_CLIENT_ID',
  'GOOGLE_CLIENT_SECRET',
  'GOOGLE_CALLBACK_URL',
  'RESEND_API_KEY'
];

console.log('=== Environment Variables Check ===');
console.log('\nRequired Variables:');
requiredVars.forEach(varName => {
  const exists = !!process.env[varName];
  console.log(`${exists ? '✅' : '❌'} ${varName}: ${exists ? 'SET' : 'MISSING'}`);
});

console.log('\nOptional Variables:');
optionalVars.forEach(varName => {
  const exists = !!process.env[varName];
  console.log(`${exists ? '✅' : '⚠️'} ${varName}: ${exists ? 'SET' : 'NOT SET'}`);
});
```

## Common Issues

### Issue: "DATABASE_URL not set"
**Solution:** Add DATABASE_URL in Render Environment tab

### Issue: "CORS error"
**Solution:** Check FRONTEND_URL is set correctly (no trailing slash, use https)

### Issue: "Google OAuth not working"
**Solution:** 
- Verify GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET
- Check GOOGLE_CALLBACK_URL matches Google Console settings
- Ensure callback URL uses https

### Issue: "Email not sending"
**Solution:** Check RESEND_API_KEY is set and valid
