How to Implement This Pattern for Other Dashboards:
Use this exact prompt/template for Client Dashboard, Recruiter Dashboard, Team Leader Dashboard, and Candidate Dashboard:

Template for Other Dashboards:
For [Dashboard Name] (Client/Recruiter/TL/Candidate):
1. *Find the main state variable* that tracks which page/tab is currently active in [Dashboard Name], similar to how admin-dashboard.tsx has:
   typescript
   const [activeTab, setActiveTab] = useState('team');
Replace it with sessionStorage restoration:

const initial[DashboardName]Tab = () => {
  const saved = sessionStorage.getItem('[dashboard-name]ActiveTab');
  sessionStorage.removeItem('[dashboard-name]ActiveTab');
  return saved ? saved : '[default-tab]';
};
const [activeTab, setActiveTab] = useState(initial[DashboardName]Tab());
Find ALL navigate calls that go to sub-pages (like /master-database, /archives, etc.)

Before EVERY navigate call, add these two lines:

sessionStorage.setItem('[dashboard-name]ActiveTab', activeTab);
navigate('/path-to-subpage');
Restart the workflow to test

## **Quick Reference Code Snippets:**
**Step 1 - Restore from sessionStorage (add near useState declarations):**
typescript
const initialActiveTab = () => {
  const saved = sessionStorage.getItem('[dashboard-name]ActiveTab');
  sessionStorage.removeItem('[dashboard-name]ActiveTab');
  return saved ? saved : '[default-tab]';
};
const [activeTab, setActiveTab] = useState(initialActiveTab());
Step 2 - Save before navigating (wrap ALL navigate calls):

onClick={() => {
  sessionStorage.setItem('[dashboard-name]ActiveTab', activeTab);
  navigate('/sub-page-route');
}}
